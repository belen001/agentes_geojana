# Skill: planificacion trazable

Objetivo: transformar una solicitud en pasos verificables.

Reglas:

- Cada decision de dominio debe tener una cita `ev-...`.
- Si falta evidencia, escribir `unknown` y crear una pregunta o tarea de investigacion.
- Separar decisiones tecnicas, riesgos y criterios de aceptacion.
- Mantener planes pequenos para reducir tokens y superficie de error.
