# Endpoints — Map Service

> Este servicio abstrae la integración con proveedores geoespaciales externos (Nominatim / OpenStreetMap) y expone operaciones de **geocodificación**, **reverse geocoding** y **validación de coordenadas**. Ningún otro servicio de dominio debe consumir directamente proveedores externos; toda operación geoespacial debe pasar por este servicio.  
> El Map Service **no** persiste datos de negocio. Sus respuestas son siempre derivadas del proveedor externo o de la validación interna, y se normalizan antes de retornarlas al solicitante.
* * *

# **Geocodificación**

> Se definen dos objetos de respuesta según la operación realizada:

### `geocodeResult` — usado en GET /geocode
Resultado de convertir un texto de búsqueda en coordenadas geográficas.

```json
{
  "latitude": "float",
  "longitude": "float",
  "displayName": "string",
  "commune": "string | null",
  "region": "string | null",
  "country": "string"
}
```

### `reverseGeocodeResult` — usado en GET /reverse-geocode
Resultado de convertir coordenadas en una dirección textual normalizada.

```json
{
  "displayName": "string",
  "address": "string",
  "commune": "string | null",
  "region": "string | null",
  "country": "string"
}
```

* * *

## **GET /geocode**
Convierte un texto de búsqueda (dirección, nombre de lugar o localidad) en una lista de resultados geoespaciales. Es utilizado por el frontend al momento de crear un reporte o un punto de interés, para que el usuario pueda buscar y seleccionar una ubicación en el mapa.

URL Params:
_Required_: `query=[string]`
_Optional_: `limit=[int]` _(por defecto: 5, máximo: 10)_

Data Params:
None

Headers:
Authorization: Bearer `<Auth Token>`
Role: usuario\_base, institucion, admin.
Content-Type: application/json

Success Response:
Code: 200
Content:

```json
{
  "results": [
    {
      "latitude": -38.7359,
      "longitude": -72.5904,
      "displayName": "Temuco, Región de La Araucanía, Chile",
      "commune": "Temuco",
      "region": "La Araucanía",
      "country": "Chile"
    },
    {
      "latitude": -38.7701,
      "longitude": -72.6115,
      "displayName": "Padre Las Casas, Región de La Araucanía, Chile",
      "commune": "Padre Las Casas",
      "region": "La Araucanía",
      "country": "Chile"
    },
    "..."
  ]
}
```

Error Response:
Code: 400
Content: `{ "error": "Query param is required" }`
OR
Code: 401
Content: `{ "error": "You are unauthorized to make this request." }`
OR
Code: 502
Content: `{ "error": "Geocoding provider unavailable" }`
* * *

## **GET /reverse-geocode**
Convierte un par de coordenadas geográficas (latitud y longitud) en una dirección textual normalizada. Es utilizado por el frontend para mostrar la dirección aproximada cuando el usuario selecciona un punto directamente sobre el mapa.

URL Params:
_Required_: `lat=[float]`
_Required_: `lng=[float]`

Data Params:
None

Headers:
Authorization: Bearer `<Auth Token>`
Role: usuario\_base, institucion, admin.
Content-Type: application/json

Success Response:
Code: 200
Content:

```json
{
  "displayName": "Avenida Alemania 0187, Temuco, Región de La Araucanía, Chile",
  "address": "Avenida Alemania 0187",
  "commune": "Temuco",
  "region": "La Araucanía",
  "country": "Chile"
}
```

Error Response:
Code: 400
Content: `{ "error": "lat and lng params are required" }`
OR
Code: 400
Content: `{ "error": "Invalid coordinate values" }`
OR
Code: 401
Content: `{ "error": "You are unauthorized to make this request." }`
OR
Code: 502
Content: `{ "error": "Geocoding provider unavailable" }`

# **Validación Geoespacial**

### `coordinateValidation` — usado en POST /validate-coordinates
Resultado de validar un par de coordenadas antes de persistirlas en el sistema.

```json
{
  "isValid": "boolean",
  "withinChile": "boolean",
  "latitude": "float",
  "longitude": "float"
}
```

> `isValid` es `false` si los valores están fuera del rango \[-90, 90\] para latitud o \[-180, 180\] para longitud, o si corresponden a coordenadas nulas (0.0, 0.0). `withinChile` es `false` si las coordenadas caen fuera del territorio continental o insular de Chile.
* * *

## **POST /validate-coordinates**
Verifica que un par de coordenadas sean geográficamente válidas y que pertenezcan al territorio de Chile. Es utilizado internamente por el Sighting Service al registrar un reporte y por el Institution Service al crear o editar un punto de interés.

URL Params:
None

Headers:
Authorization: Bearer `<Auth Token>`
Role: usuario\_base, institucion, admin.
Content-Type: application/json

Data Params:

```json
{
  "latitude": "float",
  "longitude": "float"
}
```

Success Response:
Code: 200
Content:

```json
{
  "isValid": true,
  "withinChile": true,
  "latitude": -38.7359,
  "longitude": -72.5904
}
```

Error Response:
Code: 400
Content: `{ "error": "Missing required fields" }`
OR
Code: 400
Content: `{ "error": "Invalid coordinate values" }`
OR
Code: 401
Content: `{ "error": "You are unauthorized to make this request." }`
OR
Code: 500
Content: `{ "error": "Internal server error" }`
