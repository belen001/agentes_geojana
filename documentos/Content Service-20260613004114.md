# Content Service

# Endpoints — Content Service

> Cubre dos recursos: **Especies** (catálogo de fauna) y **Protocolos de emergencia** (guía SOS). Los endpoints GET están disponibles para todos los roles autenticados. Los endpoints de escritura (POST, PATCH, DELETE) son exclusivos del rol `admin`.
* * *

# **Especies**

> Se definen dos variantes del objeto según el contexto de uso:

### `speciesList` — usado en GET /especies
Versión liviana para renderizar la cuadrícula del catálogo. No incluye los textos largos ni amenazas.

```json
{
  "id": "int",
  "commonName": "string",
  "scientificName": "string",
  "animalGroup": "mamífero | ave | reptil | otro",
  "photoUrl": "string | null"
}
```

### `speciesDetail` — usado en GET /especies/:id, POST y PATCH
Versión completa para la ficha de especie individual.

```json
{
  "id": "int",
  "commonName": "string",
  "scientificName": "string",
  "animalGroup": "mamífero | ave | reptil | otro",
  "importanceContent": "string",
  "actionContent": "string",
  "threats": ["string", "string", "..."],
  "photoUrl": "string | null"
}
```

## **GET /species**
Obtiene todas las especies del catálogo. Se puede filtrar por grupo animal para la navegación del catálogo.

URL Params:
_Optional_: `animalGroup=[mamífero | ave | reptil | otro]`

Data Params:
None

Headers:
Authorization: Bearer `<Auth Token>`
Role: usuario\_base, institucion, admin.
Content-Type: application/json

Code: 200
Content:

```json

      "animalGroup": "mamífero",

      "scientificName": "Vultur gryphus",
      "animalGroup": "ave",
      "photoUrl": "https://..."
    },
    "..."

}
```

Error Response:
Code: 401

# **Protocolos de Emergencia**

> Los protocolos tienen un many-to-many con instituciones a través de `protocolContacts`. Los contactos se gestionan directamente en el cuerpo del protocolo — no hay un endpoint separado para `protocolContacts`.

### `protocolList` — usado en GET /protocols
Versión liviana sin pasos ni contactos, para listar la guía SOS.

```json
{
  "id": "int",
  "type": "reporte | catalogo",
  "title": "string",
  "description": "string"
}
```

### `protocolDetail` — usado en GET /protocols/:id, POST y PATCH
Versión completa con pasos e instituciones de contacto expandidas.

```json
{
  "id": "int",
  "type": "reporte | catalogo",
  "title": "string",
  "description": "string",
  "steps": ["string", "string", "..."],
  "contacts": [
    {
      "id": "int",
      "name": "string",
      "emergencyPhone": "string"
    },
    "..."
  ]
}
```

> `contacts` son las instituciones vinculadas a través de `protocolContacts`, expandidas con nombre y teléfono de emergencia.

## **POST /protocols**
Crea un nuevo protocolo de emergencia. Los contactos se vinculan mediante sus IDs de institución.

URL Params:
None

Headers:
Authorization: Bearer `<Auth Token>`
Role: admin.
Content-Type: application/json

Data Params:

```json
{
  "type": "reporte | catalogo",
  "title": "string",
  "description": "string",
  "steps": ["string", "..."],
  "contactInstitutionIds ": ["int", "..."]
}
```

> `contactInstitutionIds` es la lista de IDs de instituciones a vincular como contactos del protocolo. Puede ser un array vacío.

Success Response:
Code: 201
Content: `{ <protocolDetails> }`

Error Response:
Code: 400
Content: `{ "error": "Missing required fields" }`
OR
Code: 400
Content: `{ "error": "One or more institution IDs don't exist" }`
OR
Code: 403
Content: `{ "error": "You are unauthorized to make this request." }`
OR
Code: 500
Content: `{ "error": "Internal server error" }`
* * *
